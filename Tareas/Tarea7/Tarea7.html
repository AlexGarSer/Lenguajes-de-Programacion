Símbolos

Los símbolos se refieren a objetos R. El nombre de cualquier objeto R suele ser un símbolo.
Los símbolos pueden ser creado a través de las funciones as.name y quote.
Los símbolos tienen modo "nombre", modo almacenamiento "símbolo" y tipo "símbolo". Ellos pueden ser coaccionado hacia y desde cadenas de caracteres usando as.character y as.name. Aparecen naturalmente como átomos de expresiones analizadas, intente p. as.list (cita (x + y))

Sintaxis

La sintaxis para escribir una función es:
función (arglista) cuerpo

El primer componente de la declaración de función es la función de palabra clave que le indica a R que desea crear una función. Una lista de argumentos es una lista separada por comas de argumentos formales. Un argumento formal puede ser un símbolo, una declaración de la forma "símbolo = expresión" o el argumento formal especial "...".

El cuerpo puede ser cualquier expresión R válida.
En general, el cuerpo es un grupo de expresiones contenidas en llaves (‘{’ y ‘}’).
En general, las funciones se asignan a símbolos, pero no es necesario que lo sean. El valor devuelto por la llamada a la función es una función. Si no se le asigna un nombre, se lo denomina función anónima. Las funciones anónimas se usan con mayor frecuencia como argumentos para otras funciones, como la familia de aplicación o la externa. Aquí hay una función simple: echo <- function (x) print (x). Entonces echo es una función que toma un solo argumento y cuando se invoca echo imprime su argumento.

Esqueletos

R posee un mecanismo de función genérica simple que puede usarse para un estilo de programación orientado a objetos. La distribución del método se realiza en función de la (s) clase (s) del primer argumento de la función genérica o del objeto suministrado como argumento de UseMethod o NextMethod.
Uso
UseMethod (genérico, objeto)
NextMethod (genérico = NULL, objeto = NULL, ...)
Argumentos
  • Genérico: una cadena de caracteres que nombra una función (y no un operador integrado). Necesario para UseMethod.
  • Objeto: for UseMethod: un objeto cuya clase determinará el método que se enviará. El valor predeterminado es el primer argumento de la función de cierre.


Aplicación de edición o entorno

R es un entorno de software libre para computación estadística y gráficos. Compila y se ejecuta en una amplia variedad de plataformas UNIX, Windows y MacOS. Versión actual R 4.0.0

Tipo de paradigma

R pertenece a una clase de lenguajes de programación en los que las subrutinas tienen la capacidad de modificar o construir otras subrutinas y evaluar el resultado como parte integral del lenguaje en sí.
Esto es similar a Lisp y Scheme y otros lenguajes de la variedad de "programación funcional", pero en contraste con FORTRAN y la familia ALGOL. La familia Lisp lleva esta característica al extremo mediante el paradigma "todo es una lista" en el que no hay distinción entre programas y datos. R presenta una interfaz de programación más amigable que Lisp, al menos para alguien acostumbrado a fórmulas matemáticas y estructuras de control tipo C, pero el motor es realmente muy similar a Lisp.

R permite el acceso directo a expresiones y funciones analizadas y le permite modificarlas y ejecutarlas posteriormente, o crear funciones completamente nuevas desde cero. Hay una serie de aplicaciones estándar de esta instalación, como el cálculo de derivadas analíticas de expresiones o la generación de funciones polinómicas a partir de un vector de coeficientes. Sin embargo, también hay usos que son mucho más fundamentales para el funcionamiento de la parte interpretada de R.

Algunos de estos son esenciales para la reutilización de funciones como componentes en otras funciones, como las llamadas (ciertamente no muy bonitas) a model.frame que se construyen en varias rutinas de modelado y trazado. Otros usos simplemente permiten interfaces elegantes para funcionalidades útiles. Como ejemplo, considere la función de curva, que le permite dibujar la gráfica de una función dada como una expresión como sin (x) o las facilidades para trazar expresiones matemáticas.
